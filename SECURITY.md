# Security Policy

## Where to report
- **Private Vulnerability Reporting (preferred):** Use the “Report a vulnerability” button on this repository’s Security tab.  

Please **do not** file public GitHub issues for security reports.

## Scope
- Smart contracts in this repo and their deployed instances (see Addresses in README).
- Frontend/backend components that directly affect on-chain security or user funds.

## Out of scope (examples)
- UI/UX or copy issues, rate limiting, clickjacking without impact, SPF/DMARC, non-sensitive info leaks.

## Safe Harbor
We will not pursue or support legal action against researchers who:
- Act in **good faith** and avoid privacy violations, service disruption, or data destruction.
- Report promptly and **do not disclose** details publicly until we complete remediation.
- Respect a reasonable testing scope and comply with applicable laws.

## Disclosure & Timelines
- We aim to acknowledge within **72 hours**, provide a remediation plan within **7 business days**, and publish a post-mortem (if applicable) after fixes/patches are live.
